# TÓM TẮT

## TIÊU ĐỀ

- Kiểm Tra Bảo Mật Thông Tin Cơ Bản (Basic Security Assessment)

## MỤC TIÊU BÀI THỰC HÀNH

Sau khi hoàn thành bài lab, sinh viên có thể:

1. Xác định thông tin hệ thống: phiên bản Windows, tài khoản, dịch vụ đang chạy.
2. Kiểm tra các cổng mở trên máy.
3. Kiểm tra firewall, các chính sách bảo mật cơ bản.
4. Phân tích rủi ro từ:
    - Mật khẩu yếu
    - Chia sẻ file không an toàn
    - Dịch vụ không cần thiết
5. Đề xuất một số biện pháp nâng cao an toàn hệ thống Windows.

## YÊU CẦU BÁO CÁO

Báo cáo phải bao gồm:

1. Trang bìa (tên bài, họ tên, MSSV, lớp, ngày thực hiện).
2. Mục lục.
3. Nội dung:
    - Ghi nhận quá trình thực hiện bài thực hành bằng cách mô tả các bước thực hiện, chụp ảnh màn hình các kết quả thực hiện, nêu các nhận xét và phân tích (nếu có):
        + `winver`
        + `whoami`
        + `net user`
        + `systeminfo`
        + task manager (Processes, Startup)
        + `ipconfig`
        + `netstat -ano`
        + Windows Firewall status
        + `net share`
        + Windows Update
4. Bảng tổng hợp các rủi ro phát hiện.
    - Cổng mở không cần thiết
    - Tài khoản không dùng
    - Ứng dụng lạ chạy ngầm
    - Windows chưa cập nhật
5. Biện pháp đề xuất:
    - Đổi mật khẩu mạnh
    - Tắt chia sẻ không cần thiết
    - Tắt ứng dụng startup rủi ro
    - Bật firewall đầy đủ
    - Cập nhật Windows định kỳ
6. Trả lời câu hỏi thảo luận.
7. Kết luận.
